# 丧尸末世·搜打撤生存游戏 - 设计文档

> 编写：望月 | 2026-06-03
> 引擎：Godot 4.x | 风格：俯视角像素 | 平台：PC

---

## 一、核心定位

- **类型**: 避难所经营 + 俯视角像素战术扫荡
- **参考**: Sheltered（经营层）× 丧尸末日（世界观）× 搜打撤（扫荡玩法）
- **核心张力**: 避难所生存压力驱动外出搜刮 → 搜打撤的战术决策决定能否活着回来 → 带回物资决定避难所能撑多久

## 二、核心循环

```
避难所经营
  ↓ (生存压力)
侦察/规划
  ↓
搜打撤 战术扫荡
  ├── 搜 (Search):  潜行侦察，标记目标，画路线
  ├── 打 (Strike):  快速交战（非必须，可绕）
  └── 撤 (Withdraw): 战术撤离（随时可撤）
  ↓ (带回物资 / 伤亡)
回到避难所 → 结算 → 分配 → 循环
```

## 三、搜打撤系统

### 3.1 搜（Search）
- 俯视角探图，WASD 移动
- 三种移动模式：蹲伏(静)/行走(中)/奔跑(响)
- 声音探测：跑动/战斗/交互会产生声音波 → 吸引丧尸前往调查
- 高回报区=更多丧尸护卫，低回报区=安稳但收益小
- 玩家自由规划路线，选择目标

### 3.2 打（Strike）
- 非必须，可绕。
- 近战主武器 + 远程消耗品
- 体力条限制（不能连续闪避攻击）
- 命中反馈三要素：后仰硬直 + 卡肉感 + 屏幕震动
- 开火有声音 → 引更多丧尸，形成滚雪球

### 3.3 撤（Withdraw）
- 随时可撤，不需要清图
- 三级撤离：正常(全物资) / 紧急(损失部分) / 绝望(丢物资跑)
- 撤离需要时间，伤势越重越慢
- 可选择牺牲队友掩护撤离

### 3.4 Director 系统
响应式动态难度调节，参考 L4D AI Director：
- 玩家状态好 → 加压
- 玩家快死 → 怜悯放水
- 待太久 → 丧尸潮逼近
- 不写死波次

## 四、避难所经营系统

### 资源体系
- 刚性：食物 / 水 / 药品 / 弹药（每天消耗）
- 进阶：木材 / 金属 / 电子零件 / 燃料（升级/建造）
- 特殊：情报 / 威望

### 幸存者
- 随机生成属性 + 技能（战斗/生产/社交 三类）
- 特质系统（正面/负面）
- 人际关系（好感/冲突/恋情）
- 永久死亡

### 设施树
- 生存区（优先）：床/厨房/净水器/医疗站
- 生产区（中）：工作台/种植区/回收站
- 防御区（中-高）：墙/陷阱/瞭望塔
- 特殊区（低-中）：通讯室/实验室/车库

### 事件系统
- 避难所日常事件
- 扫荡途中事件
- 全局周期性事件

## 五、技术架构

### 项目结构
```
project-game/ZombieShelter/
├── assets/sprites/characters/items/ui/effects/
├── assets/audio/sfx/music/
├── scenes/main/shelter/scavenge/characters/items/ui/
├── scripts/core/shelter/scavenge/combat/ui/
├── resources/ (items/weapons/zombies/survivors/events/maps .tres)
├── shaders/ (fog_of_war, lighting)
└── addons/
```

### 关键技术决策
- 俯视角 2D TileMap
- 实时回合混合（可暂停的实时战斗）
- 地图：预置房间模板 + 随机拼接
- AI：FSM + 感觉系统（听觉+视觉）
- 战争迷雾 + 光照
- 存档：JSON + Resource 序列化
- Tile 大小：24px

## 七、风险清单
1. 程序生成不好玩 → 先固定地图验证
2. 像素美术做不来 → 免费包占位
3. 经营层太轻 → 每天消耗硬性压力
4. 战斗手感差 → 阶段一重点 Hit Feedback
5. 搜打撤没战术感 → 丧尸 > 玩家，迫使做选择
6. 太大做不完 → 阶段一目标：5分钟封闭循环
