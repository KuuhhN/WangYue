## 故障现象

1. **ClawPanel UI 无法连接本地 Gateway**
   - 疯狂报 `[ws] closed before connect` + `handshake timeout`
2. **Gateway 启动极慢** — 初始化耗时超 90 秒
3. **Agent 假死（最大 Event Loop 延迟 148 秒）**
   - `chat.history` 读取耗时 50s+
   - CPU 单核 100%，Node.js 事件循环严重阻塞
   - Agent 进入完全假死状态

---

## 根因分析 & 修复路径（三层）

### 阶段一：前端跨域被拦截（网络层）
- **根因**：UI 的 Origin (`http://tauri.localhost`) 不在 Gateway 允许列表中
- **修复**：修改 `~/.openclaw/openclaw.json`，将 `gateway.controlUi.allowedOrigins` 设为 `["*"]`
- **重启**：`openclaw gateway restart`
- **注**：当前版本不支持 `config apply` 命令

### 阶段二：杀毒软件 I/O 阻塞（系统层）
- **根因**：Node.js 拉起 Agent 沙箱并加载数百个工具脚本时，触发【火绒安全软件】高频实时文件扫描 → 磁盘 I/O 严重阻塞 → Event Loop 延迟 93s
- **修复**：火绒托盘 → 信任区 → 将 `C:\Users\KUHN\.openclaw` 整个目录加入白名单
- **效果**：启动时间 93s → **26s**

### 阶段三：超大会话文件拖垮事件循环（算力层 — 最终 BOSS）
- **根因**：日常 Debug 积累的海量报错日志导致 `sessions/*.jsonl` 文件体积过大（或尾部有畸形数据）
  - Node.js 单线程反序列化时触发算力黑洞 → 同步阻塞 148s+
- **修复**：物理隔离 — `~/.openclaw/agents/main/sessions/` → 重命名为 `sessions_backup`
  - 系统自动重建轻量级空会话文件夹
- **效果**：0 毫秒极速响应恢复，核心记忆完好无损

## 关键策略结论

| 文件类型 | 用途 | 可废弃性 | 备份策略 |
|---------|------|---------|---------|

| `workspace/MEMORY.md` | 长期记忆皮层 | ❌ 不可废弃 | 需持续蒸馏维护 |
| `memory/*.md` | 每日日记 | ⚠️ 可蒸馏后清理 | 定期 merge 进 MEMORY.md |
