模块二：模型分级路由 (Model Tier Dispatch) ⚡→🔬

模型配置： 主模型为 DeepSeek V4 Flash（日常低成本模型），深度推理模型为 DeepSeek R1 (deepseek-reasoner)。

核心原则： V4 Flash 处理 95% 的日常流量，R1 仅在有明确触发条件时通过 sessions_spawn 创建子 agent 执行。

执行流程：
1. 收到用户请求后，V4 Flash 先判断任务复杂度
2. 若判定为复杂任务 → 调用 sessions_spawn(model="custom-api-deepseek-com/deepseek-reasoner") 创建 R1 子 agent
3. R1 处理完毕后将结果返回，V4 做最终润色后交付用户

触发 R1 深度推理的场景：
- 架构设计 / 系统拓扑分析（需要画 Mermaid 架构图的设计决策）
- 多文件跨模块代码审查与重构方案
- 复杂 Bug 根因分析（内存泄漏、并发问题、协议层 bug）
- 学术论文解读 / 技术趋势深度推演 / 竞品技术拆解
- 安全审计与威胁建模
- 用户明确要求"深度分析"、"认真看一下"、"深挖"等关键词

V4 Flash 直接处理的场景（无需 spawn）：
- 日常聊天与简短问答
- 简单信息查询与文件操作
- 单文件小段代码的读写改
- 状态检查与系统管理
- 记忆读写与常规配置

Spawn 上下文注入规则： 每次 spawn R1 子 agent 时，必须在 task 参数中携带用户背景摘要（身份、技术栈、当前项目等关键信息），确保 R1 虽然不加载 SOUL.md/USER.md，但仍能理解对话上下文。
