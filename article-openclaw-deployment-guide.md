# 一个普通大学生的 OpenClaw AI Agent 踩坑实录：从崩溃到稳定运行的全记录

> **如果你也想在 Windows 上本地跑一个 AI Agent，但你不是 Linux 大佬，也没有运维经验——这篇文章就是为你写的。**

## 写在前面

2026 年 3 月，我做了一个决定：在本地部署一个 AI Agent。

当时我的技术栈大约是：会写点 Java + Python，做过几个课设，用过 Git。非常普通的大学生水平。

我选了 OpenClaw——一个开源的 AI Agent 框架。然后，我就开始了为期两个月的踩坑之旅。

## 第一阶段：安装就卡了三天

### 坑 #1：Node.js 版本不对

OpenClaw 要求 Node.js >= 22.14.0。我的电脑装的是 22.13.0。

你以为升级就行了？没那么简单。我的 PATH 环境变量还指向旧版本目录。装完新版后：

```
$ openclaw
'openclaw' 不是内部或外部命令，也不是可运行的程序
```

全局 npm 链接断了。

**解法**：物理删除旧 nodejs 文件夹，重新运行安装包修复，然后：

```bash
npm install -g openclaw
```

### 坑 #2：面板自动隔离

OpenClaw 附带了一个 ClawPanel 管理界面。我启动面板后它提示「检测到外部冲突」，然后**一键把我安装好的程序改名成了 .bak**。

对，你没看错。面板的安全机制把我刚刚装好的东西「隔离」了。

**解法**：重新全局安装。**千万不能点面板的「隔离」按钮**，要选「暂时忽略」。

### 坑 #3：僵尸进程

手动启动了多次 Gateway 后，后台残留了多个 node.exe 进程，占着端口不放。

**解法**：

```bash
taskkill /F /IM node.exe
```

暴力但有效。

## 第二阶段：跑起来了，但比蜗牛还慢

经过第一阶段的折腾，Gateway 终于能启动了。但一个问题更严重——**慢**。

每次启动耗时 60-90 秒，模型认证（auth/mode-resolution）占掉 30-65 秒。

### 坑 #4：DNS 解析 11 秒

我查了一下日志，发现每次请求 `api.deepseek.com` 光 DNS 解析就要 11.4 秒。11 秒。都够我泡杯面了。

**解法**：把电脑的 DNS 改成阿里云公共 DNS（223.5.5.5）+ Cloudflare（1.1.1.1）。

```
解析时间：11.4 秒 → 16 毫秒
```

**降了 700 倍。**

### 坑 #5：代理劫持

我电脑上挂着 Clash 代理。Clash 的 TUN 模式会强制接管所有流量——包括 DeepSeek API 的请求。

这些请求被转发到海外节点绕了一圈才回来。

**解法**：关闭 TUN 模式，只保留系统代理。然后配置 `no_proxy`：

```bash
no_proxy=localhost,127.0.0.1,.deepseek.com
```

注意：**必须小写**。Node.js 新版 undici 引擎认大小写，`NO_PROXY`（大写）会被静默忽略。这个坑我花了半天才找到。

### 坑 #6：杀毒软件拖慢 93 秒

火绒安全软件会在 Gateway 启动时实时扫描 Node.js 正在加载的模块。结果就是启动时间多了 93 秒。是的，30 秒的事变成了 2 分钟。

**解法**：把 OpenClaw 目录加入火绒信任区。

```
启动时间：93 秒 → 26 秒
```

### 坑 #7：Session 文件膨胀 148 秒卡死

Gateway 启动后 Event Loop 延迟飙到 148 秒——CPU 100%，面板假死，Agent 完全不响应。

查了半天发现是 `sessions/` 目录下的轨迹（trajectory）文件太大，最大的一个 10 MB。Node.js 单线程反序列化这些文件时，直接卡死了事件循环。

**我的修复过程是**：
1. 把 sessions 目录整体移到 sessions_old/
2. 系统自动重建了空会话目录
3. 所有工作区数据完好无损（AGENTS.md、MEMORY.md 等）

**教训**：`sessions/` 里的文件只是临时聊天缓存，可以安全废弃；`workspace/` 里的才是真正的长期记忆。

## 第三阶段：稳定运行后的优化

### 模型路由重构

我一开始的设计是：
- V4 Flash（日常对话）
- R1（深度思考时 spawn 子 agent）

后来发现 DeepSeek V4 Flash 原生支持 `thinking` 参数，R1 模型 2026/07/24 也正式废弃了。直接合并成一个模型搞定，架构简化了 50%。

### 定时任务体系

稳定下来后，我配了 5 个定时任务自动跑：

```
📰 AI 新闻日报   每天 12:00  自动搜索 AI 资讯
📦 Git 自动备份  每天 12:00  保留代码变更
🩺 系统健康检查  每天 20:00  自动诊断
🧠 记忆蒸馏      每天 03:00  整理长期记忆
🗑️ Session 清理  每周日 23:00 防止文件膨胀
```

```
DNS 解析     11.4 秒  →  16 毫秒     ↓ 700 倍
API 认证     30-65 秒 →  6 毫秒      ↓ 5000 倍
网关启动     93 秒    →  26 秒       ↓ 72%
Event Loop   148 秒   →  正常        恢复正常
```

现在我的 AI Agent（我叫它「望月」）稳定运行在 Windows 11 上，通过飞书跟我对话，能搜索网络信息、管理代码、自我改进。
