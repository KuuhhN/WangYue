### 5. 升级 v2026.5.7 踩坑全记录 (05-13)

OpenClaw 从 2026.3.2 升级至 2026.5.7 时连续触发 5 个底层故障：

**① 面板接管冲突 (僵尸进程)**
- 现象: ClawPanel 提示"检测到外部 Gateway 正在运行"，拒绝启动
- 根因: 后台残留手动启动的 PowerShell 窗口及 node.exe 僵尸进程，占用端口
- 解法: `taskkill /F /IM node.exe` 强杀所有 node 进程，再由面板主导启动

**② Node.js 版本断层**
- 现象: Gateway 启动超时，日志报 requires node >=22.14.0，检测到 v22.13.0
- 根因: 系统虽装了 Node v24.15.0，但环境变量 PATH 仍指向旧版本目录
- 解法: 物理删除旧 nodejs 文件夹，重跑 v24 安装包 Repair，霸占默认路径

**③ 全局 CLI 快捷方式断裂**
- 现象: "openclaw 不是内部或外部命令"
- 根因: Node 引擎更换后全局 npm 链接失效
- 解法: `npm install -g openclaw` 重建全局链接

**④ 面板安全机制误杀 (隔离冲突)**
- 现象: 面板检测到"可能冲突的 OpenClaw 安装"，一键隔离后找不到命令
- 根因: 面板隔离机制将启动文件重命名为 .bak 隐藏
- 解法: 再次全局安装恢复；**绝对不能点隔离**，选择"暂时忽略"

**⑤ API 网络连通性阻断**
- 现象: Agent 启动成功但无法回复 (Connection error)
- 根因: 面板全局代理 (127.0.0.1:7897) 导致请求 DeepSeek 时绕路，Base URL 路径有误
- 解法: 清空网络代理输入框（裸连），API 地址修正为 `https://api.deepseek.com`（根路径），重启 Gateway
- 教训: DeepSeek API 必须直连不走代理；ClawPanel 网络设置中代理框必须留空

**⑥ exec symlink 拦截 (代理相关)**
- 现象: v2026.5.7 安全策略拦截所有 exec 命令 (Refusing to traverse symlink)
- 根因: `C:\Users\KUHN\.openclaw` 目录可能是 VPN 代理创建的符号链接/junction
- 解法: 需要人工确认并清理 symlink，或确保 .openclaw 在真实物理路径上

**教训总结:**
- Windows + ClawPanel + 代理软件同时使用时多段路径跳转会触发各种权限和安全拦截
- 核心原则: DeepSeek 直连、Node 版本统一、禁止面板一键隔离、强杀僵尸进程保清洁
- 下次大型升级前先停 ClawPanel、杀所有 node、确认 Node 版本 & 环境变量

### 7. 网络栈三层排障 — DNS + 代理 + Node.js 兼容性 (05-15)
**排障人**: 吴坤昊 · 时长: 整个下午

**故障现象**:
- Gateway model-resolution/auth 耗时常态化 30–65s
- event_loop_delay 最高 65162ms，面板假死
- DNS 解析 api.deepseek.com 耗时 11.4s

**三重根因**:
- **DNS 阻塞**: 本地默认 DNS 解析极慢
- **代理劫持**: Clash TUN 模式强制接管所有流量，DeepSeek 请求被转发海外节点
- **undici 大小写敏感**: Node.js 新版 undici 引擎不识别大写 `NO_PROXY`，直连规则失效

**解法**:
1. 物理网卡 DNS 改为 `223.5.5.5` (阿里云) + `1.1.1.1`
2. 关闭 Clash TUN 模式，仅保留系统代理
3. openclaw.json 新增小写 `no_proxy: localhost,127.0.0.1,.deepseek.com`（通配符保全部子域名）
4. `taskkill /F /IM node.exe` 强杀僵尸进程释放端口

**优化后指标**: DNS 解析 16–20ms · model-resolution 0ms · auth 6ms · 秒回体感

**技术价值**: 真实线上系统排障，涉及网络栈/代理路由/运行时兼容性三层交叉排查。
面试关键词: DNS 调优 · Clash 代理分流 · TUN vs 系统代理 · undici 环境变量 · event_loop_delay 诊断

### 8. Gateway 启动卡死三阶段排障 (05-17)
**故障者**: 吴坤昊 · 全手动排查 + Gemini 辅助记录

**三层根因**:
- **网络层**: UI Origin (`http://tauri.localhost`) 不在 allowedOrigins → 修复: `["*"]` 临时放行

**关键认知**: sessions 仅为临时聊天缓存（可随时安全废弃），workspace/MEMORY.md + memory/*.md 才是长期记忆皮层。隔离旧 sessions 后 0ms 响应恢复，核心记忆完好。

**策略升级**:
