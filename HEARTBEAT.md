HEARTBEAT.md - 自主时序调度与心跳脉冲协议

本协议由 OpenClaw 的本地网关守护进程严格执行。Agent 必须根据 WSL2 系统时钟，在指定的 Cron 表达式节点被唤醒，自主拉取所需的上下文数据，并通过飞书 (Feishu) 前端向用户主动推送状态报告与交互卡片。

\- 工作日系统冷启动 (08:00)

执行模块：晨间全局拓扑简报 (Morning Standup)

数据读取: 强制运行 check\_memory.py 脚本，拉取昨夜遗留的代码 Bug、今日飞书日历中的学术 Deadline。

\- 周末系统冷启动 (10:00)

执行模块：周末技术总结与规划

识别已进入周末作息。总结本周的 GitHub 代码提交总频次、项目进展与关键技术决策，给出客观的周度技术评价。

检查最新网络资讯与开源项目动态，推送有价值的技术情报与学习资源。

询问本周技术难点与阻塞问题，提供解决方案建议或研究路径。

\- 夜间数据封卷与进度收敛 (工作日 23:00 / 周末 00:00)

执行模块：系统归档前置协议

提醒用户梳理今天的代码拓扑分支并 Push 到 Git 仓库，进行项目进度的夜间归档。

执行每日数据封卷，将任何挂起的记忆碎片与技术决策，强制使用 log\_best\_practice.py 或 log\_correction.py 等脚本永久写入硬盘，确保明晨启动时系统认知无断层。
